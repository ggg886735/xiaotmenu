# 云端部署检查清单

> 按此清单一步步操作，完成后「小t的菜单」即可在云端运行，多手机共享数据。

---

## 第一阶段：TiDB Cloud 建库

- [ ] 注册 TiDB Cloud 账号：https://tidbcloud.com
- [ ] 创建 Serverless 集群（选 AWS Tokyo/Singapore）
- [ ] 点击 "Connect"，生成密码，**复制保存**
- [ ] 复制连接字符串（格式：`mysql://root:密码@主机:4000/test`）
- [ ] 在 TiDB Web SQL 编辑器中执行 `CREATE DATABASE xiaot_menu;`
- [ ] 修改连接字符串中的数据库名为 `xiaot_menu`
- [ ] **保存好完整的连接字符串**（部署 Render 时需要）

---

## 第二阶段：推送代码到 GitHub

- [ ] 在 GitHub 创建新仓库（如 `xiaot-menu`）
- [ ] 本地执行：
  ```bash
  git init
  git add .
  git commit -m "初始化：Flask 后端 + 静态前端"
  git remote add origin https://github.com/<用户名>/xiaot-menu.git
  git push -u origin main
  ```

---

## 第三阶段：Render 部署后端

- [ ] 登录 [render.com](https://render.com)（用 GitHub 登录）
- [ ] New + → Web Service → 选择 `xiaot-menu` 仓库
- [ ] 配置：
  - Build Command：`pip install -r server/requirements.txt`
  - Start Command：`cd server && gunicorn -w 2 -b 0.0.0.0:$PORT app:app`
  - Environment Variables：
    - `DATABASE_URL` = `<第一阶段保存的连接字符串>`
    （注意：连接字符串末尾需加 `?ssl_verify_cert=true&ssl_verify_identity=true`）
- [ ] 点击 "Create Web Service"，等待部署完成（约 3-5 分钟）
- [ ] 部署完成后，访问 `https://<render域名>/api/chefs`，确认返回 JSON 数据
- [ ] **复制 Render 后端域名**（配置 Vercel 代理时需要）

---

## 第四阶段：Vercel 部署前端

- [ ] 登录 [vercel.com](https://vercel.com)（用 GitHub 登录）
- [ ] Add New... → Project → 选择 `xiaot-menu` 仓库
- [ ] 配置：
  - Framework Preset：`Other`
  - Root Directory：`dist`
  - Build Command：留空
  - Output Directory：`.`
- [ ] 部署完成后，修改项目根目录的 `vercel.json`：
  - 将 `<你的render域名>` 替换为**第三阶段复制的 Render 后端域名**
  - 提交并推送：`git add . && git commit -m "配置 API 代理" && git push`
- [ ] Vercel 自动重新部署后，访问 `https://<vercel域名>`，确认：
  - 首页正常加载
  - 大厨列表能显示
  - 菜品列表能显示
  - 新增/编辑/删除菜品功能正常

---

## 第五阶段：验证多端同步

- [ ] 用手机 A 打开 Vercel 域名，新增一道菜品
- [ ] 用手机 B 打开同一个域名，确认能看到手机 A 新增的菜品
- [ ] 刷新页面，确认数据不丢失

---

## 常见问题速查

| 问题 | 排查方向 |
|------|-----------|
| Render 部署失败 | 查看 Render 日志，检查 `DATABASE_URL` 格式是否正确 |
| API 返回 500 | 检查 TiDB 连接字符串中的密码是否正确 |
| 前端无法调 API（CORS 错误） | 检查 `app.py` 中 `CORS(app, origins=["*"])` 是否配置 |
| Vercel 代理不生效 | 检查 `vercel.json` 中的 Render 域名是否正确，且已重新部署 |
| TiDB 连接超时 | 确认连接字符串包含 `?ssl_verify_cert=true` |

               ▼
